Validation Tests

Test Procedure

Validation testing is performed after development work is completed and we will plan our validating testing efforts to test the final shippable software. During our validation testing, we will test all functional aspects of the system to ensure all components are integrated well with each other and are performing cohesively as a system. We will test certain functionalities which could not have been tested without all components being ready.

Additionally, we will also focus on testing certain non-functional requirements such as performance and load bearing characteristics of the system and security aspects. For our validation testing, we will follow these testing strategies:

  • Functional Testing
  • Performance Testing
  • Load Testing

Test Cases


Functional Testing

With validation testing, we will test all functional requirements of the system and ensure that all functional requirements have been implemented according to specs.

Test Case # Test Case Description Expected Results
VTC.1

Account Creation

Following Account Creation should be performed:

  • Pet Owner Account
  • Business Owner Account
  • Business Employee Account

 Testers should be able to create these accounts with valid credentials. Duplicate accounts should not be allowed.
VTC.2

Business Account Management

Businesses should be able to perform following account management activities:

  • View details of employee accounts
  • Modify details of employee accounts
  • Create new pet records
  • Modify existing pet records

 Testers should be able to successfully perform these activities with valid business accounts.
VTC.3

User Account Management

Users should be able to perform following account management activities:

  • View details of owned pets
  • Modify details of owned pets

 Testers should be able to successfully perform these activities with valid user accounts.
VTC.4

Roles

Following security aspects should be tested w.r.t. roles

  • Business Users should not be able to create a new business account
  • Pet owners should not be able to create a business account

 Testers should NOT be able to perform these activities without system admin accounts.
VTC.5

Pet Account Maintenance

Pet owners & Business users, both should be able to perform these activities

  • Upload documentation and tie it to the pet account
  • Add/ modify events related to a pet account
  • Transfer ownership to another account

 Testers should be able to successfully perform these activities with either personal or business accounts.
VTC.6

Security

All users must enter valid credentials to access the web application.

Testers should NOT be able to login without valid accounts.

Performance Testing

With performance testing, we will try to ensure that the system is performing well under the stipulated values for response time and page load.

Test Case # Test Case Description Expected Results

Performance Testing

 Response time and latency should be under the stipulated numbers per design document.

Load Testing

With load testing, we will try to ensure that the the system performs as expected under load, and fails gracefully under stress.

Test Case # Test Case Description Expected Results

Load Testing

It is performed to ensure system performs as expected under load, and fails gracefully under stress..

 System should behave as expected.
