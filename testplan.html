Test Plan

Vision

User Feedback system will provide the platform to enable ecommerce businesses get feedback on the services and products they are offering. It will provide a comprehensive set of tools to help in all phases of a feedback loop, starting from gathering of the feedback, performing analytics on the provided feedback to informing users once the suggestion has been implemented and available for consumption. .

Scope

With User Feedback System, we will provide:

  • A unified platform where all feedback can be collected, aggregated and analyzed
  • A way to embed User Feedback forms into ecommerce web applications, to enable easy solicitation of feedback
  • Capabilities to analyze the feedback to identify most popular suggestions/issues
  • Group similar feedback to aggregate and provide meaningful insights
  • Provide outward mass communication abilities to reach out to Users who have provided feedback
  • Provide following types of feedback
    • Net Promoter Score
    • Customer Satisfaction Score
    • Customer Experience Score

Major Features

Major Feature # Major Feature Description Business Criticality
MF.1 Easy to use widget should be available to customers which they can integrate within their web application to collect feedback from users Critical
MF.2 Each customer should have a unique web portal where they could view and analyze the feedback provided by the users Critical
MF.3 A reporting system should be made available to the users which can deliver out-of-the-box and customizable reports on a configurable schedule Nice to have
MF.4 An integration point should be made available to customers which can be used to import the feedback captured from external systems and export existing feedback Must have
MF.5 The system should be able to aggregate, analyze and produce a report of most rated suggestions categorically Must have
MF.6 System should have capabilities to mass communicate with Users via email, text etc. Nice to have
MF.7 System should discrete roles built in to allow granular levels of security Must have
MF.8 Documentation/User Guide around using the System should be developed. This Documentation should be comprehensive and easy to understand/implement at the same time Must have

Not In Scope

Major features described above define the scope of the system. The features listed below are not in scope and provides clarification to the scope:

  • This system will not support non-web applications i.e. Desktop, GUI applications, mobile applications etc.
  • This system will not accept or store feedback for anonymous users.
